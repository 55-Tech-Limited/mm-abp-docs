## 此API的功能

MM 引擎自动下单、对冲头寸并跟踪盈亏。
此API让您可以：

- **监控** — 实时查看订单、对冲投注、持仓、盈亏和账户余额
- **控制** — 暂停、恢复、停止和启动您的引擎
- **配置** — 选择要交易的市场类型和锦标赛

## 基础URL

```
https://mmapi.55-tech.com
```

所有 REST 端点以 `/api/v1/` 为前缀。WebSocket 端点位于 `/ws/subscribe`。

## 关键概念

### 您的客户账户

接入时，我们为您创建客户账户并签发API密钥。该密钥用于所有 API 和 WebSocket 请求的身份验证。您通过此 API 监控引擎活动、控制交易状态，并配置要交易的市场和锦标赛。

其他所有内容 — 交易所账户、博彩公司选择、交易参数 — 均在接入阶段由我们配置。

### 博彩公司（定价源）

您的引擎连接到一个博彩公司 — 一家其实时赔率驱动您做市活动的体育博彩。当博彩公司的赔率变化时，引擎重新计算订单价格并发布到您已连接的交易所。

您不通过 API 选择或更改博彩公司，它在接入时分配。博彩公司的赔率是输入信号；交易所是您订单落地的地方。

### 交易所与预测市场

引擎代您下单的目标平台。您的交易所账户由我们的团队设置并关联至您的客户。您可通过 `GET /accounts` 查看余额和状态，账户创建、凭证和激活均由我们处理。

| 交易所 | 认证方式 | 订单类型 |
|--------|---------|---------|
| Polymarket | 以太坊钱包签名 | CLOB 限价单 |
| Polymarket US | JWT + gRPC | gRPC 订单提交 |
| Kalshi | RSA-PSS 签名 | REST 限价单 |
| Novig.us | OAuth 2.0 | REST 订单 |
| SX.bet | API密钥 + EIP712 | 签名订单 |
| Betfair | SSL 证书 | 交易所API |
| ProphetX | JWT | REST 订单 |
| Matchbook | 会话令牌 | REST 订单 |
| Smarkets | 会话令牌 | REST 订单 |
| 4casters | 令牌认证 | Socket.IO |
| Predict.fun | JWT + EIP712 | 签名订单 |

### 您控制的 vs 我们配置的

| 您控制（通过 API） | 我们配置（接入阶段） |
|-------------------|---------------------|
| 暂停 / 恢复 / 停止 / 启动您的引擎 | 您的博彩公司（定价源） |
| 取消未结订单 | 交易所账户和凭证 |
| 按运动开关市场类型 | 交易参数（限额比、价格档位） |
| 激活 / 停用锦标赛 | 客户账户创建和API密钥 |
| 监控订单、投注、持仓、盈亏 | |

### 订单状态

每个订单在 `GET /orders`、`GET /orders/open` 和 WebSocket 事件中都会返回此字段。完整枚举：

```
PENDING_PLACEMENT → PLACED → FILLED / CANCELLED / FAILED / EXPIRED
```

可使用 `?status=PLACED,FILLED` 过滤 `GET /orders`（逗号分隔多个值）。

### 匹配状态

独立于订单状态 — 跟踪订单在交易所上的撮合进度。

```
NOT_MATCHED → PARTIALLY_MATCHED → FULLY_MATCHED
```

### 结算状态

赛事结束后，标的市场结算时设置。

```
UNDECIDED → WON / LOST / VOID / HALF_WON / HALF_LOST
```

### 市场类型

市场类型自动发现。引擎持续将博彩公司的盘口映射到交易所市场 — 当引擎确认某市场类型可在您已连接的交易所交易时，该类型即可用。

**查看可用市场类型：**

```bash
curl -H "X-API-Key: YOUR_KEY" \
  https://mmapi.55-tech.com/api/v1/sports/1/market-types
```

响应中每一项包含 `marketType`、`enabled`（您当前的开关状态）和 `marketName`。使用 `PATCH /client/market-allowlist` 关闭不希望交易的类型，使用 `POST /client/market-allowlist/activate-all` 重新全部启用。

## 速率限制

- REST API：每客户端 **每分钟2,000个请求**
- WebSocket：每API密钥 **5个并发连接**
