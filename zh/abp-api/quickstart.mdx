## 步骤1：列出您的账户

检查为您的API密钥配置了哪些博彩公司账户：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  https://v2.55-tech.com/accounts
```

响应包括每个账户的博彩公司、余额、优先级、投注限额和货币：

```json
[
  {
    "client": "your-client",
    "bookmaker": "pinnacle",
    "username": "your-username",
    "balance": 5000.00,
    "active": true,
    "priority": 10,
    "maxStake": 1000.00,
    "minStake": 5.00,
    "currencyId": "USD"
  }
]
```

## 步骤2：获取投注单

下注前，获取当前赔率和限额：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  "https://v2.55-tech.com/betslip?fixtureId=YOUR_FIXTURE_ID&outcomeId=161&playerId=0"
```

投注单返回每个有此市场的博彩公司的实时赔率：

```json
{
  "pinnacle": {
    "price": 1.85,
    "limit": 5000.00,
    "limitMin": 5.00,
    "limitCurrency": "USD",
    "active": true,
    "bookmakerFixtureId": "123456",
    "bookmakerMarketId": "789",
    "bookmakerOutcomeId": "1"
  },
  "betfair_ex": {
    "price": 1.88,
    "limit": 2000.00,
    "limitMin": 2.00,
    "limitCurrency": "GBP",
    "active": true
  }
}
```

<Note>
使用 [OddsPapi API](https://docs.oddspapi.io/) 查找赛事ID和结果ID。ABP 直接使用 OddsPapi 标识符。
</Note>

## 步骤3：下单

下一个带最低价格保护的投注订单：

```bash
curl -X POST \
  -H "X-ABP-API-KEY: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "orders": [
      {
        "fixtureId": "id1000000861624412",
        "outcomeId": 161,
        "playerId": 0,
        "orderStake": 50,
        "orderPrice": 1.80,
        "orderCurrency": "USD",
        "userRef": "my-strategy-1",
        "acceptBetterOdds": true,
        "acceptPartialStake": true,
        "back": true,
        "testOrder": false
      }
    ]
  }' \
  https://v2.55-tech.com/place-orders
```

**关键字段：**

| 字段 | 必填 | 描述 |
|------|------|------|
| `fixtureId` | 是 | OddsPapi 赛事ID |
| `outcomeId` | 是 | 市场结果（例如 161 = 主胜） |
| `playerId` | 是 | 非球员盘设为 `0` |
| `orderStake` | 是 | 总投注金额 |
| `orderPrice` | 是 | 最低可接受十进制赔率 |
| `orderCurrency` | 是 | 货币代码（例如 `USD`） |
| `userRef` | 否 | 用于分组相关订单的引用 |
| `acceptBetterOdds` | 否 | 接受优于 `orderPrice` 的赔率（默认：true） |
| `acceptPartialStake` | 否 | 允许跨多个投注的部分成交（默认：true） |
| `back` | 否 | 买入注（true）或卖出注（false）（默认：true） |
| `testOrder` | 否 | 仅验证，不实际下注（默认：false） |
| `expiresAt` | 否 | ISO 8601 过期时间（默认：从现在起5秒） |

**响应：**

```json
{
  "orders": [
    {
      "orderId": 12345,
      "requestUuid": "eb45b192-358b-32d5-9f65-af497b9fa8c1",
      "orderStatus": "PENDING",
      "fixtureId": "id1000000861624412",
      "outcomeId": 161,
      "orderStake": 50,
      "orderPrice": 1.80,
      "filledStake": 0,
      "bets": []
    }
  ],
  "declinedOrders": []
}
```

验证失败的订单会出现在 `declinedOrders` 中，并包含 `declineReason`：

```json
{
  "declinedOrders": [
    {
      "fixtureId": "...",
      "outcomeId": 161,
      "declineReason": "Order stake 150.00 USD exceeds available limit 100.00 USD"
    }
  ]
}
```

## 步骤4：跟踪您的订单

检查订单状态：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  https://v2.55-tech.com/orders
```

查看为您的订单下的单个投注：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  https://v2.55-tech.com/bets
```

每个投注包括博彩公司、下注价格、下注金额、投注状态和结算状态：

```json
{
  "betId": 1,
  "orderId": 12345,
  "bookmaker": "pinnacle",
  "bookmakerBetId": "3587791057",
  "placedStake": 50.00,
  "placedPrice": 1.85,
  "placedCurrency": "USD",
  "betStatus": "CONFIRMED",
  "settlementStatus": "UNSETTLED",
  "account": "your-username",
  "userRef": "my-strategy-1"
}
```

## 步骤5：查看持仓和盈亏

查看您的汇总敞口：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  https://v2.55-tech.com/positions
```

查看盈亏：

```bash
curl -H "X-ABP-API-KEY: YOUR_API_KEY" \
  https://v2.55-tech.com/pnl
```
