## 端点

```
wss://v2.55-tech.com/ws
```

## 连接流程

### 1. 连接

打开WebSocket连接。连接时无需身份验证。

### 2. 登录

在连接后 **30秒** 内发送登录消息：

```json
{
  "type": "login",
  "apiKey": "your-api-key",
  "channels": []
}
```

空的 `channels` 数组订阅所有可用频道。要订阅特定频道：

```json
{
  "type": "login",
  "apiKey": "your-api-key",
  "channels": ["orders", "bets", "settlements"]
}
```

### 3. 登录确认

成功后，服务器响应：

```json
{
  "type": "login_ok",
  "clientName": "your-client",
  "channels": ["orders", "bets", "settlements"],
  "subscriptionId": 1,
  "access": {
    "clientFiltered": ["orders", "bets", "settlements", "accounts", "balance"],
    "global": ["fixtures", "currencies", "status", "emergency"]
  },
  "features": {
    "messageOrdering": true,
    "acknowledgments": false,
    "batchAck": false
  }
}
```

### 4. 接收更新

数据消息格式如下：

```json
{
  "type": "data",
  "channel": "orders",
  "event": "INSERT",
  "payload": { ... },
  "ts": 1771183909789,
  "seq": 1
}
```

| 字段 | 描述 |
|------|------|
| `type` | 数据消息始终为 `"data"` |
| `channel` | 此消息所属频道 |
| `event` | 事件类型：`INSERT`、`UPDATE`、`DELETE`、`STATUS_CHANGE`、`SETTLED` |
| `payload` | 完整的更新对象 |
| `ts` | 服务器时间戳（自纪元以来的毫秒数） |
| `seq` | 用于消息排序的序列号 |

### 5. 保持活跃

每 **30秒** 发送一次心跳：

```json
{"type": "ping"}
```

服务器响应：

```json
{"type": "pong"}
```

服务器也会每 **60秒** 发送心跳：

```json
{"type": "heartbeat", "ts": 1771183909789}
```

## 频道

### 客户端过滤频道

这些频道仅传递属于您 `clientName` 的数据：

| 频道 | 事件 | 描述 |
|------|------|------|
| `orders` | INSERT, UPDATE, STATUS_CHANGE | 订单下达、成交、状态变更 |
| `bets` | INSERT, UPDATE, SETTLED | 投注下达、确认、结算 |
| `settlements` | UPDATE | 结算状态更新 |
| `accounts` | UPDATE | 账户余额和状态变更 |
| `balance` | UPDATE | 余额变动通知 |

### 全局频道

所有订阅者都会收到：

| 频道 | 事件 | 描述 |
|------|------|------|
| `fixtures` | UPDATE | 赛事元数据和比分变更 |
| `currencies` | UPDATE | 货币汇率更新 |
| `status` | UPDATE | 系统状态变更 |
| `emergency` | UPDATE | 紧急模式激活/停用 |

## 连接限制

| 设置 | 值 |
|------|-----|
| 每API密钥最大连接数 | 5 |
| 认证超时 | 30秒 |
| 服务器心跳间隔 | 60秒 |
| 客户端心跳间隔 | 30秒（推荐） |
| 消息缓冲区大小 | 每订阅100条 |

## 示例：Python 客户端

```python
import asyncio
import json
import websockets

async def connect():
    uri = "wss://v2.55-tech.com/ws"
    async with websockets.connect(uri) as ws:
        # 登录
        await ws.send(json.dumps({
            "type": "login",
            "apiKey": "YOUR_API_KEY",
            "channels": ["orders", "bets", "settlements"]
        }))

        # 等待登录确认
        login_resp = json.loads(await ws.recv())
        print(f"已登录为 {login_resp.get('clientName')}")

        # 保活任务
        async def keep_alive():
            while True:
                await asyncio.sleep(30)
                await ws.send(json.dumps({"type": "ping"}))

        asyncio.create_task(keep_alive())

        # 监听更新
        async for message in ws:
            data = json.loads(message)
            if data["type"] == "data":
                print(f"[{data['channel']}:{data['event']}] {data['payload']}")

asyncio.run(connect())
```

## 示例：JavaScript 客户端

```javascript
const WebSocket = require('ws');

const ws = new WebSocket('wss://v2.55-tech.com/ws');

ws.on('open', () => {
  ws.send(JSON.stringify({
    type: 'login',
    apiKey: 'YOUR_API_KEY',
    channels: ['orders', 'bets', 'settlements']
  }));
});

ws.on('message', (raw) => {
  const msg = JSON.parse(raw);

  if (msg.type === 'login_ok') {
    console.log(`已登录为 ${msg.clientName}`);
  }

  if (msg.type === 'data') {
    console.log(`[${msg.channel}:${msg.event}]`, msg.payload);
  }
});

// 保持活跃
setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify({ type: 'ping' }));
  }
}, 30000);
```
