## 包含内容

**ABP API** 文档涵盖：

- **账户管理** — 跨30多家集成的博彩公司账户完整CRUD操作
- **投注单** — 通过 OddsPapi v5 获取实时赔率和限额，支持多博彩公司聚合
- **下单** — 智能路由，支持幂等性（`requestUuid`）、部分成交和可配置过期时间
- **投注跟踪** — 基于游标的分页查询，从 PENDING 到 CONFIRMED/SETTLED 的完整生命周期
- **分析** — 按博彩公司、账户或用户引用分组的持仓和盈亏
- **WebSocket** — 订单、投注、结算、余额、赛事、货币、状态、紧急事件的实时推送
- **30+博彩公司集成** — pinnacle、betfair-ex、polymarket、kalshi、cloudbet、smarkets 等

**MM API** 文档涵盖：

- 做市引擎配置和实时价差管理
- WebSocket驱动的定价更新和订单簿交互

**Scraping API** 文档涵盖：

- **HTTP 抓取** — 通过 80 多个地理分布式代理节点进行 GET/POST/PUT/PATCH/DELETE 请求
- **WebSocket 中继** — 支持 Socket.IO、SignalR、Centrifugo、GraphQL-WS、原生 WS 的双向帧中继
- **AMQP 消费者** — 通过 SSE 流式传输 RabbitMQ 消息
- **网络情报** — 代理节点列表、地理分布、按域名健康检查
- **使用量与速率限制** — 按密钥的指标、请求分析、热门域名

## 推荐用法

**支持获取URL的AI助手：**

- `/llms-full.txt` — 最适合"一次性阅读全部内容"（所有指南页面+示例）
- `/abp-api/openapi.json` — ABP 端点和模式准确性（带类型和示例）
- `/mm-api/openapi.json` — MM 端点和模式参考
- `/scraping-api/openapi.json` — Scraping 端点和模式参考

**复制/粘贴工作流：**

在浏览器中打开 `/llms-full.txt` 并粘贴内容到您的AI工具中。

**代码生成：**

将 OpenAPI 规范提供给 `openapi-generator`、`oapi-codegen` 或 Cursor/Copilot 等工具，用于生成类型化客户端SDK。

## 说明

- `llms-full.txt` 遵循与网站导航相同的顺序
- WebSocket 协议在指南页面中记录；REST 端点在 OpenAPI 规范中
- 所有三个 API（ABP、MM、Scraping）包含在同一个文件包中
- 所有 API 通过 `X-API-Key` 请求头进行身份验证
